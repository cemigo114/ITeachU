# 6.EE.B.5 · Grade 6 · Expressions and Equations
## "The Key to the Code"
*Use substitution to determine if a value makes an equation or inequality true.*

---

## 1. Student Prompt
"I found a locked digital safe that requires a 'secret value' for $x$ to open. The screen shows this code: $3x + 4 = 19$. I have a small bag of numbered keys: $\{3, 4, 5, 6\}$. My friend says I should just start turning the dial, but I think I can test each key without touching the safe. If I put the '5' key into the $x$ spot, will the math sentence tell me it's true ($19 = 19$), or am I going to get a 'Wrong Key' error message? How do I know for sure which key is the winner?"

---

## 2. Possible Misconceptions

1. **The "Answer is the Solution" Error** *(Type: procedural_overgeneralization)*
Students often think "solving" means performing an operation on the numbers they see (like $19 + 4 = 23$) rather than finding a value for $x$ that balances the statement.

2. **Inequality Boundary Confusion** *(Type: partial_concept_model)*
In a problem like $x + 2 > 7$, a student might test the value 5 and see $7 > 7$. They might think 5 is a solution because it "hits the target," failing to realize that "greater than" means the result must actually be larger, making the statement false for $x=5$.

3. **Multiplication vs. Concatenation** *(Type: surface_feature_matching)*
When substituting 5 into $3x$, a student might read it as "thirty-five" ($35 + 4$) instead of $3 \times 5 + 4$. This leads them to discard the correct key because the math seems impossible.

---

## 3. Pattern Recognition Prompts

- "If you try the '4' key, you get $3(4) + 4$. Is that $12 + 4 = 16$? Is 16 the same as 19, or do you need a 'heavier' key?"
- "Wait—if the screen said $x < 10$ and your keys were $\{8, 9, 10, 11\}$, would more than one key be 'true'? Or can there only ever be one winner?"
- "When you swap the letter for a number, does the equals sign ($=$) act like a judge deciding if the two sides are perfectly matched?"

---

## 4. Generalization Question (Always / Sometimes / Never)
"I have a hunch that for an EQUATION (with an $=$ sign), there is ALWAYS only one correct key in the world... but for an INEQUALITY (with $>$ or $<$), there could be MANY correct keys. Is that always the case, or could an equation have two winners?"

---

## 5. Inference and Prediction
"I found a second lock that says $2x > 10$. If our keys are still $\{3, 4, 5, 6\}$, which keys will actually work this time? If we found a key labeled '5.1', would that be a winner too, even though it's not a whole number?"

**Prediction target:** Identifying that only $\{6\}$ works for $2x > 10$ among the whole numbers, and recognizing that rational numbers (like 5.1) can also be solutions to inequalities ($10.2 > 10$).

---

## 6. Mapping and Process Data

### Claims
- Students can define a solution as a value that makes an algebraic statement true.
- Students can use substitution to test values from a given set in both equations and inequalities.

### Evidence
Evidence is gathered when a student substitutes $x=5$ into $3x + 4 = 19$, calculates $15 + 4 = 19$, and identifies it as the "true" solution. The task forces this by providing a specific "set" of keys, moving the focus from calculation to the logic of verification.

### Process Data Revealing Critical Thinking
A student demonstrates deep understanding when they can explain that the "solution" is the value that makes the statement a "fact." Self-correction is evident when a student tries $x=6$ for $3x+4=19$, gets 22, and realizes the value is "too big," logically concluding they need to try a smaller key from their set.
