# 8.F.A.1 · Grade 8 · Functions
## "The Vending Machine Glitch"
*Understand that a function is a rule that assigns to each input exactly one output. The graph of a function is the set of ordered pairs consisting of an input and the corresponding output.*

---

## 1. Student Prompt
I was using the school vending machine and noticed that every time I press button A1, a bag of pretzels comes out, but when my friend pressed button B2 yesterday, a water bottle came out, and today it gave him a granola bar. Is this machine still following a "rule," or is it just doing whatever it wants now?

---

## 2. Possible Misconceptions

1. **One-to-One Requirement** *(Type: partial_concept_model)* The student believes that a function must have a unique output for every input, thinking that two different buttons (A1 and A2) both giving pretzels would "break" the rule.

2. **Input-Output Reversal** *(Type: structural_misunderstanding)* The student thinks that because they can get pretzels from two different places, the machine isn't working, failing to see that the "rule" only cares about what happens when you push a specific button.

3. **Predictability Confusion** *(Type: misinterpretation_of_representation)* The student mistakes a consistent pattern (A1 always gives pretzels) for a function even if other inputs (B2) are broken, struggling to define the "rule" for the whole system.

---

## 3. Pattern Recognition Prompts

- If you pushed button A1 a hundred times in a row, what would have to happen for you to be 100% sure the "A1 rule" is working?
- When your friend pushed B2, did the machine "know" exactly what to give him, or was it like rolling a die?
- How does the "button" you pick connect to the "snack" you get in a way that someone else could predict?

---

## 4. Generalization Question (Always / Sometimes / Never)
"Wait, so for a machine to be a 'function,' does it mean that if I pick a specific input, it ALWAYS has to give me the exact same result every single time, even if other inputs also give that same result?"

---

## 5. Inference and Prediction
I found a different machine where button C3 and button C4 both give you the same brand of apple juice. My friend says that's also a "glitch," but I think it's fine as long as C3 doesn't suddenly start giving out orange juice tomorrow. Do you think a graph of this "apple juice" machine would look like a straight line, or would the dots be all over the place?

**Prediction target:** Whether a single input (button) having multiple possible outputs (snacks) prevents a relationship from being a function.

---

## 6. Mapping and Process Data
### Claims
- The student can define a function as a rule where each input is assigned exactly one output.
- The student can identify non-functions
